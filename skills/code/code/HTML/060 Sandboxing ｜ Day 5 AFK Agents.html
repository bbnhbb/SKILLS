    Sandboxing

    Running an AFK agent without permission requests means giving it a lot of freedom. That's risky. Without supervision, an agent can delete entire directories, exfiltrate sensitive code, or break your system in ways that take hours to fix.

    The solution is Sandcastle. Sandcastle runs agents inside isolated sandboxes where they can only touch files you've explicitly shared. The agent can still explore code, run bash commands, and work with git, but it's confined to that sandbox. Permission requests become unnecessary because the agent cannot harm the rest of your system.

    This is safer than trying to restrict agents with rules alone. Agents can break out of software sandboxes if they try hard enough. Sandcastle supports multiple sandbox types, including Docker containers and E2B sandboxes, which work at the operating system level - they're much harder to escape.

    Steps To Complete

    Install Docker Desktop or Podman

    • Download Docker Desktop from https://docs.docker.com/get-started/get-docker/

    Choose the right version for your operating system (Mac, Windows, or Linux).

    Alternatively, you can use Podman, which is free and open source: https://podman.io/. Docker Desktop is free but has licensing constraints for large organizations.

    • Install Docker Desktop (or Podman) and launch it

    Make sure the application is running before moving to the next step.

    Configure Your Sandcastle Environment

    • Navigate to the .sandcastle directory in your project

    • Open the .env.example file

    You'll see environment variables that need to be added to a .env file. These variables authenticate your coding agent inside the Docker container.

    • Create a .env file in the .sandcastle directory

    Copy the contents from .env.example and fill in your authentication details:

    ANTHROPIC_API_KEY=your-api-key-here

    Use your Anthropic API key for authentication. Get one from https://console.anthropic.com/.

    Using your Claude subscription is possible. Check here for up-to-date advice.

    • Save the .env file

    You don't need to worry about the GitHub token yet.

    Build the Docker Image

    • Open your terminal and navigate to your project root

    • Run the command to build the Docker image:

    pnpm sandcastle docker build-image

    This builds the Docker image for your repository. Sandcastle uses the Dockerfile in the .sandcastle directory, which sets up Node.js 22, installs the agent, and creates a container environment.

    The build might take a couple of minutes on first run.

    Test the Sandbox

    • Run the test file to verify everything works:
    npx tsx .sandcastle/test.ts

    This runs a simple AFK agent with the prompt "hello, how are you?" inside your Sandcastle container.

    • Watch the logs appear in your terminal

    You should see agent logs showing up. Look for a link you can click (command-click or control-click) to view live logs.

    • Check the live logs

    The logs should show:

    • Agent started
    • Agent's response (e.g., "hello, I'm doing well, thanks for asking, how can I help you today?")
    • How long sandbox setup took
    • How many commits were made (should be zero for this simple test)
    • How much context window was used

    If you see a successful run with these details, Sandcastle is properly configured and ready to run AFK agents autonomously.
