    Setting Up And Trying AFK Agents

    You've learned how agents work in human-in-the-loop mode. You've set up the Docker sandbox that makes autonomous execution possible. Now it's time to experience the real goal: watching your agent run completely independently.

    This is where all the pieces come together. The same PRD, the same agents setup, the same feedback loops from earlier, but now your agent executes without needing your decisions. You're going to start the run, watch the terminal stream in real-time, and see autonomous development actually happen.

    The key moments you'll notice: commands executing without approval gates, tests and type checks running automatically, files changing and accumulating. This is your agent doing what you asked, at full speed, without steering.

    Steps To Complete

    Prepare and Review

    • Review the script you'll be running

    Look at the .sandcastle/main.ts file. This is the main entry point for your AFK agent run. It takes your PRD and plan as positional arguments instead of asking for them interactively.

    Start Your AFK Run

    • Run the AFK agent command with your PRD and plan

    Using npx tsx, execute the main script and pass your PRD and plan file paths as arguments:

    npx tsx .sandcastle/main.ts "prd/admin-analytics.md" "plans/admin-analytics-plan.md"

    The agent will start running immediately. Output will stream to your terminal as it works.

    • Understand what's different from interactive mode

    The script uses run mode with two key settings:

    • maxIterations: 3 - The agent will loop a maximum of 3 times
    • completionSignal: '<promise>NO MORE TASKS</promise>' - If the agent outputs "promise no more tasks", it stops early

    This means the agent will keep working until either it runs 3 times or it declares there are no more tasks to complete.

    • Watch for these key moments as the agent runs:

    Sandbox permissions working: You'll see bash commands executing without any approval prompt. The agent can run code, write files, execute tests, all autonomously.

    Feedback loops in action: Tests and type checks run automatically as the agent works. These are the concepts you learned before, now running without human intervention.

    File changes accumulating: Watch as the agent edits files, iterates, and builds. The same quality work you saw in HITL mode, just faster and uninterrupted.

    • Let the run complete

    When the agent finishes or reaches the iteration limit, the terminal will show the completion message.

    Validate What the Agent Built

    • Compare the output against your original PRD

    Did the agent build what you asked for? Look at the files it created, the changes it made, the tests it wrote. Is this aligned with your PRD?

    • Notice what's different from HITL mode

    In HITL mode, you made decisions mid-run. The agent asked you questions. Here, none of that happened. The agent made autonomous choices and kept moving.

    Document Your Experience

    • Note any moments where the agent did something unexpected

    Did it make a choice you wouldn't have made? Did it recover from an error? Did the feedback loops catch something?

    • If you hit any issues, head to the Discord

    Setup problems? Agent getting stuck? Share what you're seeing and we'll help debug.
