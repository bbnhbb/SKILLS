    Hooking Up Agents To Your Backlog

    Commits
    07.06.01

    Start the lesson — prompt is wired to GitHub issues from the previous lesson

    07.08.01

    See my solution — DevUI user selector bug fixed via AFK agent

    You've set up your fork of the cohort project. Now it's time to use an agent to automatically find and fix a real bug in the codebase.

    There's a small but annoying issue in the dev UI: when you select a user from the user selector dropdown, the dropdown doesn't close. It just stays open. Everyone in the cohort has complained about it at some point.

    In this lesson, you'll create an issue describing this bug, set up your Docker image, and let an AFK agent automatically fix it.

    Steps To Complete

    Create and Document the Bug

    • Navigate to your forked repository on GitHub

    Go to the issues section and create a new issue.

    • Write an issue description for the bug

    Describe the problem clearly. Here's an example of how you might write it:

    User selector in the dev UI panel does not close after selecting a user. It's annoying, fix it.

    Keep it straightforward - the agent will read this and understand what needs to be fixed.

    Build Your Docker Image

    • Inside your repository, build a new Docker image for your fork

    Run this command in your terminal:

    pnpm sandcastle docker build-image

    The first time you run this, it will take a moment to build. Since you're working in a new repository fork, it needs to create a fresh image with a new name. It will build from sandcastle-cohort-004-project to something specific for your fork.

    Run the AFK Agent

    • Execute the agent using tsx

    Once your Docker image is built, run the agent with this command:

    npx tsx .sandcastle/main.ts

    The agent will:

    • Pick up the issue you created
    • Locate the bug in your codebase
    • Fix the bug automatically
    • Close the issue when complete

    Watch as the agent does the work for you. This is the power of autonomous coding agents.

    Verify the Fix

    • Check that the issue was closed

    Return to your GitHub issues page. The issue you created should now be marked as closed.

    • Review the changes the agent made

    Look at the recent commits in your repository to see exactly how the agent fixed the bug. This helps you understand what was wrong and how it was corrected.

    • Test the bug fix in the dev UI

    Start your dev server and navigate to the user selector. Select a user like Sarah Chen and verify that the dropdown now closes properly after selection.
