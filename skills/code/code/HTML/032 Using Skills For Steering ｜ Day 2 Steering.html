    Using Skills For Steering

    Commits
    04.05.01

    Start here - added the /write-a-skill skill to the repo

    04.05.02

    See my solution — /coding-standards skill added

    You now understand what skills are and how they work. The best way to learn how to write a skill? Use a skill to write a skill.

    We're going to take the coding standards currently scattered throughout CLAUDE.md and reorganize them into a proper skill. Right now, every request to your agent includes all hundred-plus lines from that file. That's wasteful context.

    A coding standards skill would let you make these guidelines available when needed, without bloating every single request.

    Steps To Complete

    • Open your agent and activate the /write-a-skill skill

    This skill will guide you through building your new coding standards skill.

    • Write a detailed description of what you want to create

    Your description should explain:

    • You want to extract all the coding standards from CLAUDE.md

    • You want to organize them into progressively disclosed sections

    • Different parts of the coding standards should be easy for the LLM to find

    • The standards shouldn't be pushed into the context by default (they should only load when needed)

    • Have a conversation with the agent to develop your skill

    The conversation here is crucial. This is where you'll develop your ability to produce high-quality skills. Ask clarifying questions. Iterate on the structure. Think about how the LLM will actually use these standards.

    • Review the generated skill in the solution

    Once you're done, compare your approach with the provided solution to see what patterns work well.

    • Delete the coding standards from CLAUDE.md

    After creating your skill, you can remove the redundant content and leave just a reference to your new skill.
